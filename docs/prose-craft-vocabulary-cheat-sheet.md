# Prose-Craft Vocabulary Cheat Sheet

Working reference for tightening **Editor**, **Author (Narrator)**, and **Worker** system prompts.  
**Author = Narrator** in this codebase — the IC prose GM (`AUTHOR_SYSTEM_PROMPT` in [`src/prompts.ts`](../src/prompts.ts)).

Only terms mapped to current prompts or observed logs. Uncertain items flagged at the end.

---

| **Register**                            | The diction level, genre conventions, and emotional temperature of prose — what it _sounds like_, not what happens. | Worldbook `Register:` field; story-to-date "same register, third person"; OUTPUT PROSE blocks.                | "Couched to match the **tonality** of the content and setting" → "Match the **Register** line in CONTENT — diction level, heat, and genre conventions."                                                              | Worldbook and memory blocks share one explicit tone anchor instead of fuzzy "tonality." |

### Author POV & tense — likely needs separate prompts

**Play-tested pattern (main haunt story):** user posts in **third person** (`Lex …`); Author replies in **second person** from Lex's POV (`you …`). The model drifts — collapses to one person, or flips **past vs present** — even without an explicit user mode change. Tense drift plausibly comes from `[STORY TO DATE]` (always third-person **past**) overwhelming unstated IC tense preference. A full scene can pass before drift is noticed; retroactive consistency isn't always worth it.

**Current prompt gap:** [`AUTHOR_SYSTEM_PROMPT`](../src/prompts.ts) allows both "PC approaches" and "I approach" player inputs and says "respond from the PC's perspective" but never locks **Author person** or **tense**. Memory pipeline assumes Author _may_ use second person ([`worldbook-pc.ts`](../src/services/worldbook-pc.ts) converts `you/your` → PC name in summaries).

**Recommendation:** treat POV as a **per-story setting**, not a one-size Author paragraph. Options (not mutually exclusive):

| Mode                 | Player posts                        | Author narration        | Notes                                                                               |
| -------------------- | ----------------------------------- | ----------------------- | ----------------------------------------------------------------------------------- |
| **Split (tabletop)** | Third (or first) — player writes PC | Second — "you see/hear" | User's haunt-story preference; needs **tense discipline** hard against memory past. |
| **Unified third**    | Third                               | Third — `Lex sees …`    | Matches memory person; less cognitive split for some players.                       |
| **Unified second**   | Second / "I"                        | Second                  | Classic GM; matches much RP training data.                                          |

Implementation sketch (for later — not in cheat-sheet scope):

1. **CONTENT field** e.g. `POV: split | third | second` and `Tense: present | past` set at Editor setup.
2. **`AUTHOR_SYSTEM_PROMPT` variants** (or injected block) selected from that field — cleaner than one prompt trying to cover all modes.
3. **`IC_PROSE_STEERING`** repeats POV + tense every generation (like Register), explicitly: "Memory is third past; your IC voice is not."

Do **not** fold POV into **filter words** or **head-hopping** rules — those apply _within_ a chosen mode.

## Guided continue / retry

Bracket-delimited OOC steering ([`src/prompts.ts`](../src/prompts.ts) `guidedRegenerateNote`, `guidedContinueNote`, `icProseSteeringNote`; wired in [`pipeline-runner.ts`](../src/queue/pipeline-runner.ts)).

| Path                                   | What gets appended                                                                  | Craft note                                                                                           |
| -------------------------------------- | ----------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **Author** continue/retry              | `[IC_PROSE_STEERING …]` + optional user guidance                                    | User guidance merges into same bracket block — treat as **OOC** directive, not prose to continue.    |
| **Editor** setup/update continue/retry | `[Continue the conversation…]` / `[Take the following into special consideration…]` | No `IC_PROSE_STEERING`; no pulp/register override.                                                   |
| **Story-to-date seam retry**           | `buildSeamRetryUserMessage()` — roll back coverage                                  | "Same **register** and rules as before" — reuse Editor **Register** + **telling-not-showing** terms. |

**Why brackets:** Plain-sentence instructions were observed to bleed into generated prose; closing `]` is a hard stop (comment in `prompts.ts` lines 212–219).

**Mood toggles** ([`toggle-presets.ts`](../src/services/toggle-presets.ts)): "visceral, sensory" / "lighter, less heavy" — **disabled** in `applyGenerationOptions` (line 131). If re-enabled, map to **eyeball kicks** / **purple prose** (intense) and **pacing** + anti-**purple prose** (light).
