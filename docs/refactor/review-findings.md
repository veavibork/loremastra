### [x] src/queue/dispatch.ts:414

**error-handling** · slice `be-queue-core` · verdict: confirmed

resolveHordeJob's status.done branch (fillTextGeneration/getPage/maybeQueueStoryNameJob/finishJob/publishDone) has no try/catch, so any exception there leaves the job permanently stuck in 'running' with no error surfaced.

**Failure scenario:** scanHordeJobs calls `void resolveHordeJob(...)` fire-and-forget for every running Horde job on each 500ms tick. Inside resolveHordeJob, only the pollTextGeneration call is wrapped in try/catch (lines 393-405); everything after `if (status.done)` (getText/getPage/fillTextGeneration/maybeQueueStoryNameJob/getBookByType/maybeEnqueueStoryToDateJob/finishJob/publishDone, lines 414-436) is unguarded.
